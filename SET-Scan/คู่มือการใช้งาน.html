ทำความรู้จักโปรแกรม

SET Dashboard คือโปรแกรมสำหรับ วิเคราะห์ภาพรวมตลาดหุ้นไทย ช่วยให้คุณเห็นข้อมูลสำคัญในหน้าเดียว ไม่ว่าจะเป็น:

  • ภาพรวมตลาด — กี่ % ของหุ้นทั้งหมดกำลังขึ้น
  • กลุ่มอุตสาหกรรมไหนแข็งแกร่งที่สุด
  • หุ้นที่กำลัง Breakout หรือทำ New High
  • กรองหุ้นตามเงื่อนไขที่กำหนดเอง
  • ติดตามหุ้นที่สนใจใน Watchlist
โปรแกรมนี้เหมาะสำหรับ: นักลงทุนที่ต้องการ Top-Down Analysis — มองภาพใหญ่ก่อน แล้วค่อยลงมาหาหุ้นเดี่ยว

การอัปเดตข้อมูล

โปรแกรมมีปุ่มอัปเดต 2 แบบที่มุมบนขวา ใช้ต่างกันตามสถานการณ์:

ปุ่มใช้เมื่อระยะเวลาสิ่งที่ทำ
⚡ Quick Update ใช้ทุกวัน (หลังทำ Full Refresh ครั้งแรกแล้ว) ~2–5 นาที ดาวน์โหลดเฉพาะวันที่ขาดไป + recalculate metrics ทั้งหมด (ราคา, EMA, RS, ATH)
⟳ Full Refresh ครั้งแรกที่ติดตั้ง หรือต้องการดึง Fundamentals ใหม่ (P/E, P/BV, Div) 20 นาที – 1.5 ชั่วโมง ดาวน์โหลด history ใหม่ทั้งหมดตามช่วงที่เลือก + Fundamentals ทุกตัว
ลำดับการใช้ครั้งแรก: ต้องกด ⟳ Full Refresh ก่อนเสมอ เพื่อสร้าง set_history.json (ไฟล์เก็บประวัติราคา) หลังจากนั้นกด ⚡ Quick Update ทุกวันได้เลย ไม่ต้อง Full Refresh อีก

เลือก Period สำหรับ Full Refresh

เมื่อกด ⟳ Full Refresh จะมีหน้าต่างให้เลือก ช่วงเวลาที่ต้องการดาวน์โหลด

Periodประวัติที่ได้เวลาโดยประมาณแนะนำสำหรับ
2Yย้อนหลัง 2 ปี~20–30 นาทีเครื่องที่ต้องการประหยัดเวลา / ทดสอบ
5Yย้อนหลัง 5 ปี~30–45 นาทีใช้งานทั่วไป กราฟ 5Y จะมีข้อมูลครบ
10Yย้อนหลัง 10 ปี~45–60 นาทีต้องการดู EMA200 แม่นยำสูง
Maxตั้งแต่ IPO (สูงสุดถึงปี 2001)~1–1.5 ชั่วโมงข้อมูลสมบูรณ์ที่สุด ATH แม่นยำที่สุด
แนะนำ Max สำหรับครั้งแรก — ได้ประวัติยาวที่สุด EMA200 แม่นยำที่สุด และ ATH (All-Time High) ถูกต้อง 100% หลังจากนั้นใช้ Quick Update ทุกวันแทน

คำศัพท์สำคัญ

ก่อนใช้งานควรทำความเข้าใจคำศัพท์พื้นฐานเหล่านี้ก่อนครับ

RS Score (Relative Strength Score)
คะแนน 0–99 วัดว่าหุ้นตัวนี้ แข็งแกร่งกว่าตลาดแค่ไหน คำนวณจากผลตอบแทนย้อนหลังหลายช่วงเวลาเทียบกับหุ้นทั้งหมดใน SET

80–99 แข็งแกร่งมาก (Top 20% ของตลาด)   50–79 ปานกลาง   0–49 อ่อนแอกว่าตลาด
RS Score = Rank percentile ของ (1M×2 + 3M×1 + 6M×1 + 1Y×1)
EMA (Exponential Moving Average)
ค่าเฉลี่ยราคาย้อนหลังแบบถ่วงน้ำหนัก ใช้ดูแนวโน้ม

EMA 20 = แนวโน้มระยะสั้น (ประมาณ 1 เดือน)
EMA 50 = แนวโน้มระยะกลาง (ประมาณ 2.5 เดือน)
EMA 200 = แนวโน้มระยะยาว (ประมาณ 10 เดือน)

ราคา อยู่เหนือ EMA = แนวโน้มขาขึ้น / ราคา อยู่ใต้ EMA = แนวโน้มขาลง
52W High / 52W Low
ราคาสูงสุดและต่ำสุดในช่วง 52 สัปดาห์ (1 ปี) ที่ผ่านมา

หุ้นที่ราคา ทำ 52W High ใหม่ = กำลังอยู่ในแนวโน้มขาขึ้นแข็งแกร่ง
หุ้นที่ราคา ใกล้ 52W Low = อาจเป็นโอกาสถ้าสัญญาณกลับตัวชัดเจน
Market Breadth
วัดว่ามีหุ้นกี่ % ที่ราคาอยู่เหนือเส้น EMA ต่างๆ

ถ้า % สูง เช่น 70% อยู่เหนือ EMA50 = ตลาดโดยรวมอยู่ในแนวโน้มขาขึ้น
ถ้า % ต่ำ เช่น 30% อยู่เหนือ EMA50 = ตลาดโดยรวมอ่อนแอ
Sec.Rank (Sector Rank)
อันดับของหุ้นในกลุ่ม sector เดียวกัน เรียงจาก RS Score สูงสุด

เช่น 2/15 = อันดับ 2 จาก 15 หุ้นในกลุ่มเดียวกัน

อันดับ 1–25% หุ้นแข็งแกร่งในกลุ่ม   25–50% ปานกลาง   >50% อ่อนกว่าครึ่งกลุ่ม
P/E (Price to Earnings)
อัตราส่วนราคาต่อกำไร — ตลาดยินดีจ่ายกี่เท่าของกำไรต่อปี

< 15x ถูก   15–25x ปานกลาง   > 25x แพง (หรือกำไรต่ำมาก)
P/BV (Price to Book Value)
อัตราส่วนราคาต่อมูลค่าทางบัญชี — ตลาดยินดีจ่ายกี่เท่าของสินทรัพย์สุทธิ

< 1x ซื้อได้ต่ำกว่ามูลค่าทางบัญชี   1–2x ปานกลาง   > 2x แพงกว่ามูลค่าบัญชี
Dividend Yield (อัตราผลตอบแทนเงินปันผล)
ปันผลที่ได้รับต่อปี คิดเป็น % ของราคาหุ้นปัจจุบัน

≥ 4% ปันผลสูง   2–4% ปานกลาง   < 2% ต่ำ
RVOL (Relative Volume — ปริมาณซื้อขายสัมพัทธ์)
วัดว่าวันนี้มีปริมาณซื้อขาย มากหรือน้อยกว่าค่าเฉลี่ย 20 วันกี่ %

> 200% ปริมาณสูงผิดปกติ (มักเกิดเมื่อมีข่าวหรือ Breakout)   100–200% สูงกว่าปกติ   < 100% ต่ำกว่าค่าเฉลี่ย
RVOL = (Volume วันนี้ / ค่าเฉลี่ย Volume 20 วัน) × 100
ATH (All-Time High — ราคาสูงสุดตลอดกาล)
ราคาสูงสุดของหุ้นตัวนั้นตลอดช่วงเวลาที่มีข้อมูล (สูงสุดถึงปี 2001)

ATH% = ระยะห่างจาก ATH เป็นเปอร์เซ็นต์ ค่าติดลบเสมอ (เว้นแต่กำลังทำ ATH ใหม่)

≥ −5% ใกล้ ATH มาก — อาจกำลัง Breakout ใหม่   −5% ถึง −20% ยังอยู่ไม่ไกล   −20% ถึง −40% ห่างจาก ATH พอสมควร   < −40% ต่ำกว่า ATH มาก
ATH% = (ราคาปัจจุบัน − ATH) / ATH × 100

Stat Cards (กล่องตัวเลขด้านบน)

ตัวเลขความหมาย
หุ้นทั้งหมดจำนวนหุ้น SET ทั้งหมดในระบบ
ขึ้นวันนี้จำนวนหุ้นที่ราคาเพิ่มขึ้นจากวันก่อน
ลงวันนี้จำนวนหุ้นที่ราคาลดลงจากวันก่อน

ความหมายของแต่ละโซน

โซนสีการตีความ
ต่ำกว่า -2SDเขียวเข้มUndervalued มาก — ถูกมาก เทียบกับประวัติศาสตร์ของหุ้นตัวนั้น
-2SD ถึง -1SDเขียวอ่อนUndervalued — ถูกกว่าค่าเฉลี่ย
-1SD ถึง AVGเหลืองต่ำกว่าค่าเฉลี่ยเล็กน้อย — Fair Value ค่อนข้างถูก
AVG ถึง +1SDส้มสูงกว่าค่าเฉลี่ย — Fair Value ค่อนข้างแพง
+1SD ถึง +2SDแดงอ่อนOvervalued — แพงกว่าค่าเฉลี่ย
สูงกว่า +2SDแดงเข้มOvervalued มาก — แพงผิดปกติ ควรระวัง
หมายเหตุ: SD Band วัดความถูก/แพง เทียบกับตัวเองในอดีต ไม่ใช่เทียบกับหุ้นตัวอื่น หุ้นที่ปกติ P/E สูง (เช่น Growth Stock) อาจอยู่ใกล้ AVG แม้ P/E จะสูงกว่าตลาด
วิธีใช้ร่วมกับ RS Score: หุ้นที่ Valuation อยู่ใต้ AVG + RS Score ≥ 60 = โอกาสที่น่าสนใจ เพราะทั้งถูกและเริ่มมี Momentum ขึ้น

💰 กลยุทธ์ที่ 2: หาหุ้นปันผลสูง + RS ดี

เหมาะสำหรับนักลงทุนที่ต้องการ กระแสเงินสดสม่ำเสมอ พร้อมกับหุ้นที่มีแนวโน้มราคาดีด้วย

หลักการ: หุ้นปันผลสูงที่ RS ดีด้วย = ได้ทั้ง Passive Income และ Capital Gain — ดีกว่าเลือกแค่ปันผลสูงอย่างเดียวซึ่งอาจเป็นหุ้นที่ราคาตกนาน

ระวัง Yield Trap: บางหุ้น Div Yield สูงเพราะราคาตกลงมามาก ไม่ใช่เพราะปันผลเยอะขึ้น ดูด้วยว่า RS Score ดีด้วยหรือเปล่า ถ้า RS ต่ำมาก (< 30) ควรระวัง

หุ้นที่ขาดทุน (กำไรติดลบ) P/E จะแสดงเป็น N/A สีแดง ถ้าแสดงเป็น — หมายความว่า Yahoo Finance ไม่มีข้อมูล (พบบ่อยในหุ้นขนาดเล็กหรือ REIT)

RS Score คำนวณยังไง?

คำนวณจากผลตอบแทน 1M×2 + 3M×1 + 6M×1 + 1Y×1 แล้ว Rank เป็น Percentile ในกลุ่มหุ้นทั้งหมด คะแนน 99 = ดีที่สุด 1% ของตลาด

EMA200 ในกราฟ Max ใช้ข้อมูลย้อนหลังนานแค่ไหน?

ใช้ประวัติราคาตั้งแต่ที่ yfinance มีให้ — สำหรับหุ้น SET ส่วนใหญ่ย้อนหลังได้ถึง ปี 2001-2004 (ประมาณ 20+ ปี) EMA200 จึงแม่นยำมากเพราะมี Warmup period เพียงพอ

Val. Band ดึงข้อมูลจากไหน?

ดึงจาก mrlikestock.com ครั้งแรกที่ค้นหา แล้ว Cache ไว้ 6 ชั่วโมง ในหน่วยความจำ Server การค้นหาครั้งต่อไปภายใน 6 ชั่วโมงจะโหลดจาก Cache ทันที ไม่ต้องรอ ถ้า Cache หมดอายุหรือ restart server จะดึงข้อมูลใหม่จาก mrlikestock.com อีกครั้ง

Watchlist หายหลังเปิด Browser ใหม่?

Watchlist เก็บใน localStorage ของ Browser — ถ้าล้าง Cache หรือเปิด Incognito จะหาย แนะนำใช้ Browser เดิมเสมอ

เปิดจากคอมพิวเตอร์เครื่องอื่นในบ้านได้ไหม?

ได้ครับ เปิดจากเครื่องที่รัน start.bat แล้วเข้าเว็บด้วย IP ของเครื่องนั้น เช่น http://192.168.1.x:5000

ไฟล์ set_history.json ใหญ่มาก ปกติไหม?

ปกติครับ ไฟล์นี้เก็บประวัติราคา + ปริมาณซื้อขายของหุ้น ~900 ตัว ย้อนหลัง 20+ ปี ขนาดประมาณ 60–90 MB ไม่ได้โหลดใน browser จึงไม่กระทบความเร็วของ Dashboard
